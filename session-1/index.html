01 — activations exist for a reason

A straight line can't wrap a ring.
One ReLU layer can.

ClaimA model with no nonlinearity can only draw a straight boundary, so it cannot separate two interleaved rings — adding one ReLU hidden layer can.
Build~300 noisy 2D points as two rings. Train (a) a single linear layer + sigmoid, (b) one ReLU hidden layer (16 units). Same data, same optimizer.
ProofWatch both decision boundaries form live — the linear one stays a straight line stuck near chance, the ReLU one wraps the ring.

STEP 01

Generate the data

Two rings — inner is class 0, outer is class 1. No straight line separates them by construction.

STEP 02

Fit a linear model

One layer plus sigmoid can only carve space with one straight edge — against a ring, that's a coin flip.

STEP 03

Add one ReLU layer

Each hidden unit folds space once. Stack enough folds and the combination bends into a closed loop.

STEP 04

Compare the boundaries

Same update rule, same data. The boundary picture is the proof — one is a line, the other a ring.

02 — depth without nonlinearity is a lie

Stack five linear layers.
You still only get one line.

ClaimFive stacked linear layers collapse to a single linear map — a 5-layer linear net is no stronger than 1 layer. Both fail the ring task the same way. Insert ReLUs between the same five layers and it suddenly solves it.
BuildSame ring data. Train 1 linear layer, 5 linear layers (no activations), then the same 5 layers + ReLU.
ProofThe 1-layer and 5-linear-layer boundaries both stay straight lines, capped at the same low ceiling. ReLU breaks the tie. Bonus: multiply the five weight matrices numerically — the product really is just one matrix.

03 — embeddings learn similarity from nothing but next-token

Nobody tells the model which words
are related. It figures it out anyway.

ClaimTrained only to predict the next token in a tiny synthetic grammar, the embedding table clusters related tokens — even though similarity was never supplied.
BuildA toy language: animals (cat, dog, cow), fruits (apple, mango), verbs (eat, chase, see), generated as animal → verb → fruit sentences. A tiny embedding → softmax next-token model learns purely to predict what comes next.
ProofThe learned embeddings are already 2D — no projection needed. Same-category tokens land together, and every token's nearest neighbor is from its own category.

04 — memorization vs generalization

Same model, same capacity.
The only thing that changes is how much data it sees.

ClaimA high-capacity model on tiny data drives train accuracy to ~100% while held-out accuracy stays behind — growing the dataset closes that gap.
BuildA noisy ring classification with a fixed held-out test set. Train the same over-parameterized ReLU network (64 hidden units) three times, on 20, 200, and 2000 training points.
ProofAt n=20 the boundary is jagged and memorizes the exact points; at n=2000 it's smooth and generalizes. The train/test gap shrinks as the dataset grows.
